# dev-platform Roadmap

Spec sequence for the `teelr/dev-platform` repo. Each entry is a Roadmap Phase — its detailed Spec lives under `tasks/`.

- **R1: Foundation** *(complete — 2026-05-08, `tasks/dev-platform-foundation-spec.md`)* — repo restructure, top-level dir contracts, install / uninstall / verify scripts. Migrated `~/.claude/{commands,skills/WORKFLOW_MANUAL.md,settings.json}` into the repo as the source of truth; live cutover executed against `~/.claude/`.
- **R1.5: Global Claude + Hooks Coverage** *(complete — 2026-05-09, `tasks/dev-platform-global-claude-and-hooks-spec.md`)* — closes the gateway-policy gap: tracks `~/.claude/CLAUDE.md` as `dev/settings/claude-global.md`, ships first hook script (`hooks/post-tool-heartbeat.sh`) plus the `settings.json` hooks-block wiring. 13 symlinks now flow from `dev/` → `~/.claude/`. The heartbeat is the data source R2 will consume.
- **R4a: Project Scaffolding** *(complete — 2026-05-10, `tasks/dev-platform-r4a-scaffolding-spec.md`)* — three starter templates (`go-service`, `python-agent`, `next-frontend`) under `scaffolding/`, orchestrator at `scripts/new-project.sh`, conversational Q&A pattern at `docs/NEW-PROJECT.md`, Scope-rule scaffolding carve-out inline in `dev/CLAUDE.md`. Assistant runs the scaffold; user describes the project. R4b (VSCode user-profile + extensions list) deferred to its own spec.
- **R3: Testing** *(complete — 2026-05-10, `tasks/dev-platform-r3-testing-spec.md`)* — `scripts/gate_fast.sh` orchestrator + 5 auto-discovered test suites under `tests/` (hook fixtures, command frontmatter validator, taxonomy self-test, install round-trip, scaffold smoke). 42 mechanical checks aggregate to a single PASS/FAIL in < 1s. Replaces conversation-derived gate-fast with a runnable script every spec from here on rides. Auto-discovery: any new `tests/<suite>/*.sh` is picked up without editing the orchestrator. Deferred to future small specs: `gate_full.sh` (per-template builds), pre-commit git hook, performance benchmarks.
- **R2: Monitoring** *(NEXT — planned)* — telemetry aggregation on top of the R1.5 heartbeat data: gate pass rate, `/code` retry counts, `/review` catch rate, hook execution time per project. Schemas in `monitoring/`, additional collectors as hook scripts in `hooks/`. R3 was promoted ahead of R2 to land first because every spec needs gate coverage to ship safely; that's now in place, so R2 telemetry is the next natural extension.
- **R4b: VSCode Coverage** *(planned)* — VSCode user-profile config (settings.json, keybindings, snippets), statusline scripts, tracked `extensions/vscode/extensions.json` extensions list with sync helper. Deferred from R4 because the remote-SSH client/server split deserves its own design pass.
- **R5: Migration tooling** *(planned)* — auto-migrate older project layouts (legacy taxonomy, scattered config) onto the standard project structure described in `CLAUDE.md`.

Canonical roadmap detail (per-Roadmap-Phase exit criteria, dependencies, demo definition) lives in `tasks/dev-platform-roadmap.md` — created when R2 starts.
